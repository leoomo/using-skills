# Skill Optimization Checklist

## 触发准确性优化检查清单

### Description 字段检查

- [ ] 以 "Use when" 开头
- [ ] 描述触发条件，不总结工作流程
- [ ] 包含具体症状/场景关键词
- [ ] 包含用户可能使用的短语
- [ ] 第三人称描述
- [ ] 长度 <500 字符（理想 <300）

### 触发测试

| 测试类型 | 示例查询 | 预期结果 |
|----------|----------|----------|
| 正例-标准 | "optimize this skill" | 触发 |
| 正例-口语 | "帮我改进一下这个技能" | 触发 |
| 正例-问题 | "skill triggering wrong" | 触发 |
| 负例-相似 | "create a new skill" | 不触发 |
| 负例-无关 | "write python code" | 不触发 |

### 常见触发问题

| 问题 | 症状 | 修复方案 |
|------|------|----------|
| 描述过于抽象 | 该触发时不触发 | 添加具体场景关键词 |
| 描述过于具体 | 不该触发时触发 | 扩大范围或添加排除条件 |
| 缺少同义词 | 部分表达不触发 | 添加常见同义表达 |
| 与其他技能竞争 | 触发不稳定 | 明确区分边界 |

## Token 效率优化检查清单

### 内容检查

- [ ] 无重复段落
- [ ] 示例精简（一个好例子 > 多个平庸例子）
- [ ] 大段内容已拆分到引用文件
- [ ] 流程图仅在复杂决策时使用
- [ ] 表格比列表更高效的地方用了表格

### 行数目标

| 技能类型 | 目标行数 | 备注 |
|----------|----------|------|
| 频繁加载 | <100 行 | 每次 conversation 都加载 |
| 常用技能 | <200 行 | 经常触发 |
| 专用技能 | <400 行 | 特定场景 |
| 复杂技能 | 拆分 + 引用 | 超过 400 行考虑拆分 |

### Token 节省技巧

1. **引用文件**：大段参考内容放到 `references/`
2. **精简示例**：一个完整好例子 > 多个不完整例子
3. **合并重复**：相同模式用表格统一
4. **删除废话**：移除 "请注意"、"请注意" 等填充词
5. **压缩格式**：用表格替代多行列表

## 指令清晰度优化检查清单

### MUST/ALWAYS/NEVER 使用检查

- [ ] MUST/ALWAYS 数量 <5 个（过多 = 信号稀释）
- [ ] 每个 MUST 都有明确的 WHY 解释
- [ ] NEVER 有替代方案说明
- [ ] 没有相互矛盾的指令

### 歧义检查

- [ ] 没有 "可能"、"也许"、"考虑" 等模糊词
- [ ] 边界情况有明确示例
- [ ] 技术术语有上下文解释
- [ ] 数量词有具体值（"2-3 个" 而非 "一些"）

### 清晰度改进模板

**改进前：**
```markdown
如果可能的话，尝试优化性能。
```

**改进后：**
```markdown
优化性能。优先考虑：
1. 减少循环嵌套层级
2. 使用缓存避免重复计算
3. 批量处理替代逐个处理
```

## 结构优化检查清单

### 章节组织

- [ ] Overview 在最前面
- [ ] 核心流程在主体
- [ ] 参考内容在末尾或引用文件
- [ ] 有快速参考表格（如适用）

### 流程图使用

| 场景 | 是否需要流程图 |
|------|----------------|
| 线性步骤 | ❌ 用编号列表 |
| 有分支决策 | ✅ 用流程图 |
| 有循环/迭代 | ✅ 用流程图 |
| 有多种入口 | ✅ 用流程图 |

### 引用文件组织

```
skill-name/
├── SKILL.md           # 主文件，<300 行
├── references/
│   ├── api-docs.md    # API 参考
│   ├── examples.md    # 详细示例
│   └── troubleshooting.md  # 故障排除
└── scripts/
    └── helper.py      # 可执行脚本
```
